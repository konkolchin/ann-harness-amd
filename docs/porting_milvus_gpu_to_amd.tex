\begin{abstract}
This document describes a layered experiment plan for replacing NVIDIA cuVS/cuRAFT with
AMD hipVS/hipRAFT in the Milvus/Knowhere GPU path. It complements the CPU benchmarking
runbook (\texttt{ann\_framework\_runbook.tex}): FAISS and Milvus Standalone CPU results on
\texttt{amd-rx7900xtx} are the control; GPU port work proceeds in five layers from ROCm sanity
through standalone libraries, Knowhere, Milvus source builds, and harness benchmarks.
\end{abstract}

\section{Background}

Milvus does not call cuVS directly from Python. The stack is:

\begin{verbatim}
pymilvus / VectorDBBench
        |
   Milvus server
        |
   Knowhere  (vector engine: CPU + GPU indexes)
        |
   GPU path: GPU_IVF_FLAT, GPU_IVF_PQ, GPU_CAGRA, GPU_BRUTE_FORCE
        |
   NVIDIA: RAFT / cuVS
   AMD goal: hipRAFT + hipVS (ROCm-DS ports)
\end{verbatim}

\textbf{hipVS} is an AMD port of cuVS
(\url{https://github.com/ROCm-DS/hipVS},
\url{https://rocm.docs.amd.com/projects/hipVS/en/latest/}) with API compatibility as an
explicit goal. \textbf{hipRAFT} is the matching primitives layer (RAFT/cuRAFT analogue on
ROCm).

The stock \textbf{Milvus v2.5.4 CPU Docker image} used for CPU baselines does \emph{not}
exercise the GPU/HIP path. GPU experiments require \textbf{source builds} (or a custom GPU
image you build yourself). NVIDIA Milvus GPU images will not run on AMD.

\section{Prerequisites (CPU control)}

Before starting GPU layers, the following should be in place on the target AMD host
(see the main runbook for commands and measured tables):

\begin{itemize}[leftmargin=*]
  \item SIFT-1M HDF5 dataset and in-repo FAISS harness (\texttt{run\_faiss\_hdf5.py}).
  \item Milvus Standalone Docker CPU (\texttt{milvus-docker/}, \texttt{user.yaml}
        \texttt{knowhere.simdType: sse4\_2}).
  \item Optional but recommended: VectorDBBench \texttt{milvusivfflat} IVF \texttt{nprobe}
        sweep for same-host FAISS vs Milvus CPU comparison.
\end{itemize}

On \texttt{amd-rx7900xtx} (7900 XTX / \texttt{gfx1100}), CPU baselines and the IVF sweep
were captured 2026-06-30. GPU work does not replace those numbers; it adds a new backend
to compare against them.

\section{Layered strategy}

Do not attempt to ``replace cuVS in Milvus'' in one step. Use \textbf{five layers}
(Layer~0 through Layer~4), each with explicit pass/fail criteria. Only advance when the
current layer passes.

\centering

\begin{tabular}{@{}clp{0.52\textwidth}@{}}
\toprule

\section{Layer 1 --- hipRAFT and hipVS in isolation}
\label{sec:layer1-hipvs-hipraft}

\subsection{Goal}

Prove AMD libraries work on the GPU \textbf{before} Knowhere or Milvus. Layer~1 success means:
\texttt{libcuvs.so} builds for \texttt{gfx1100}, hipVS tests run on GPU, and IVF/brute-force search
returns sensible neighbors.

\subsection{Reality check (7900 XTX / RDNA3)}

\begin{itemize}[leftmargin=*]
  \item \textbf{RDNA3 / \texttt{gfx1100} requires warp size 32.} hipCollections (cuco) defaults to
        CDNA-style warp size 64; without \texttt{USE\_WARPSIZE\_32=ON} configure fails with
        \texttt{not supported with warp size 64 (gfx1100)}. This path is \textbf{experimental}
        in upstream cuco.
  \item Official build docs ask for \textbf{ROCm~$\geq$7.0.2} and \textbf{cmake~$\geq$3.31}. The lab
        host has ROCm~\texttt{6.4.4} today---try Layer~1 as-is; if configure fails on missing
        packages, upgrade ROCm to the newest \texttt{6.4.x}/\texttt{7.x} line AMD documents for
        hipVS branch \texttt{release/rocmds-25.10}, then rebuild.
  \item First build downloads dependencies via ROCmDS-cmake and can take \textbf{1--3+ hours} and
        \textbf{10--30\,GB} disk under \texttt{\textasciitilde{}rocmds\_check\_gfx1100/}.
\end{itemize}

\subsection{Step 1 --- Host packages (once)}

On \texttt{amd-rx7900xtx}, inside \texttt{tmux}:

\begin{verbatim}
sudo apt update
sudo apt install -y git ninja-build ccache \
  build-essential pkg-config \
  libopenblas-dev libblas-dev liblapack-dev libsuitesparse-dev

# Ubuntu 22.04 ships cmake 3.22; hipVS wants >= 3.31
python3 -m pip install --user 'cmake>=3.31'
export PATH="$HOME/.local/bin:$PATH"
cmake --version   # must show >= 3.31

# ROCm tools on PATH
export PATH=/opt/rocm/bin:$PATH
export ROCM_PATH=/opt/rocm
hipcc --version
/opt/rocm/bin/rocminfo | grep -E 'gfx1100|7900'
\end{verbatim}

Optional: add user to GPU groups if builds/tests fail with permission errors:

\begin{verbatim}
sudo usermod -aG render,video "$USER"
# log out and back in
\end{verbatim}

\subsection{Step 2 --- Clone pinned sources}

Use branch \texttt{release/rocmds-25.10} for both repos (matched pair):

\begin{verbatim}
export WORKDIR=~/rocmds_check_gfx1100
export INSTALL_PREFIX=$WORKDIR/install
mkdir -p "$WORKDIR" "$INSTALL_PREFIX"

cd "$WORKDIR"
git clone -b release/rocmds-25.10 --depth 1 https://github.com/ROCm-DS/hipRaft.git
git clone -b release/rocmds-25.10 --depth 1 https://github.com/ROCm-DS/hipVS.git
\end{verbatim}

\subsection{Step 2b --- Patch rocThrust version for ROCm 6.4.x}

hipRaft branch \texttt{release/rocmds-25.10} overrides rocThrust to \textbf{4.0.0} (ROCm~7+).
ROCm~6.4.4 apt ships \textbf{3.3.x}. Without this patch, CMake tries to download rocThrust and
may hit a broken URL (\texttt{https://github.com//rocThrust/}).

\begin{verbatim}
python3 <<'PY'
import json
from pathlib import Path
p = Path.home() / "rocmds_check_gfx1100/hipRaft/versions.json"
data = json.loads(p.read_text())
data.setdefault("packages", {}).setdefault("rocthrust", {})["version"] = "3.3.0"
p.write_text(json.dumps(data, indent=2) + "\n")
print("patched", p)
PY
\end{verbatim}

\subsection{Step 3 --- Environment for builds}

\begin{verbatim}
export PATH=$HOME/.local/bin:/opt/rocm/bin:$PATH
export ROCM_PATH=/opt/rocm-6.4.4
export WORKDIR=~/rocmds_check_gfx1100
export INSTALL_PREFIX=$WORKDIR/install
export PARALLEL_LEVEL=$(nproc)

export CMAKE_PREFIX_PATH=$ROCM_PATH:$ROCM_PATH/lib/cmake:$INSTALL_PREFIX/lib/cmake
export HIP_DIR=$ROCM_PATH/lib/cmake/hip
export rocthrust_DIR=$ROCM_PATH/lib/cmake/rocthrust
export rocprim_DIR=$ROCM_PATH/lib/cmake/rocprim
export hipcub_DIR=$ROCM_PATH/lib/cmake/hipcub
export RAPIDS_CMAKE_ROCTHRUST_USE_LOCAL=ON
export RAPIDS_CMAKE_ROCM_ORG=ROCm
export RAPIDS_CMAKE_ROCM_DS_ORG=ROCm-DS
\end{verbatim}

Do \textbf{not} pass \texttt{--cmake-args=-D...} (bash strips the inner quotes; \texttt{build.sh}
then fails \texttt{Invalid option}). For \texttt{gfx1100}, wrap the \textbf{entire} flag in single
quotes so literal \texttt{"..."} reach \texttt{build.sh} (Step~4).

\subsection{Step 4 --- Build hipRAFT first}

hipVS depends on hipRAFT. Install both into \texttt{\$INSTALL\_PREFIX}:

\begin{verbatim}
cd $WORKDIR/hipRaft
./build.sh clean && rm -rf cpp/build _deps
# --gpu-arch MUST be last. Single-quote the whole --cmake-args flag (bash quote trap).
INSTALL_PREFIX=$INSTALL_PREFIX ./build.sh libraft tests --compile-lib \
  --cache-tool=ccache \
  '--cmake-args="-DUSE_WARPSIZE_32=ON"' \
  --gpu-arch="gfx1100"
\end{verbatim}

\textbf{Pass:} exit code~0; libraries under \texttt{\$INSTALL\_PREFIX/lib} (e.g.\ \texttt{libraft.so}).

If configure complains about missing \texttt{hipblas}/\texttt{hipsparse}/\texttt{rocthrust}, install ROCm dev packages and export the \texttt{*_DIR} variables from Step~3 (do not pass them via \texttt{--cmake-args}):

\begin{verbatim}
sudo apt install -y rocm-dev rocm-hip-sdk rocthrust-dev rocprim-dev hipcub-dev

# verify system rocThrust (must exist before hipRAFT configure)
ls /opt/rocm/lib/cmake/rocthrust/rocthrust-config.cmake \
   /opt/rocm-6.4.4/lib/cmake/rocthrust/rocthrust-config.cmake 2>/dev/null

export PATH=$HOME/.local/bin:/opt/rocm/bin:$PATH
export ROCM_PATH=/opt/rocm-6.4.4   # or /opt/rocm if configs live there
export CMAKE_PREFIX_PATH=$ROCM_PATH:$ROCM_PATH/lib/cmake:$INSTALL_PREFIX/lib/cmake
export rocthrust_DIR=$ROCM_PATH/lib/cmake/rocthrust
export rocprim_DIR=$ROCM_PATH/lib/cmake/rocprim
export hipcub_DIR=$ROCM_PATH/lib/cmake/hipcub
export RAPIDS_CMAKE_ROCTHRUST_USE_LOCAL=ON
export RAPIDS_CMAKE_ROCM_ORG=ROCm

cd $WORKDIR/hipRaft
./build.sh clean && rm -rf cpp/build _deps
INSTALL_PREFIX=$INSTALL_PREFIX ./build.sh libraft tests --compile-lib \
  --cache-tool=ccache \
  '--cmake-args="-DUSE_WARPSIZE_32=ON"' \
  --gpu-arch="gfx1100"
\end{verbatim}

If you still see \texttt{Build step for rocthrust failed} or \texttt{github.com//rocThrust/},
patch \texttt{versions.json} (Step~2b), confirm \texttt{ls \$rocthrust\_DIR} succeeds, then wipe \texttt{cpp/build \_deps} and retry.

\subsection{Step 5 --- Build hipVS (libcuvs + focused tests)}

Start with a \textbf{small test subset} before compiling all gtests:

\begin{verbatim}
cd $WORKDIR/hipVS
./build.sh clean
INSTALL_PREFIX=$INSTALL_PREFIX CMAKE_PREFIX_PATH=$CMAKE_PREFIX_PATH \
  ./build.sh libcuvs tests --cache-tool=ccache \
  '--cmake-args="-DUSE_WARPSIZE_32=ON"' \
  --gpu-arch="gfx1100" \
  --limit-tests="NEIGHBORS_ANN_IVF_FLAT_TEST;BRUTEFORCE_C_TEST"
\end{verbatim}

Full test suite (much longer):

\begin{verbatim}
INSTALL_PREFIX=$INSTALL_PREFIX CMAKE_PREFIX_PATH=$CMAKE_PREFIX_PATH \
  ./build.sh libcuvs tests \
  '--cmake-args="-DUSE_WARPSIZE_32=ON"' \
  --gpu-arch="gfx1100"
cd $WORKDIR/hipVS/cpp/build
ctest --test-dir ./tests --output-on-failure
\end{verbatim}

\textbf{Pass:} \texttt{libcuvs.so} installed; \texttt{NEIGHBORS\_ANN\_IVF\_FLAT\_TEST} and/or
\texttt{BRUTEFORCE\_C\_TEST} exit~0.

\subsection{Step 6 --- Run tests manually}

\begin{verbatim}
export LD_LIBRARY_PATH=$INSTALL_PREFIX/lib:$ROCM_PATH/lib:$LD_LIBRARY_PATH
GTEST=$WORKDIR/hipVS/cpp/build/gtests

$GTEST/BRUTEFORCE_C_TEST

\section{Layer 2 --- Knowhere with HIP backend}

\subsection{Goal}

Substitute the library link at the Knowhere layer, still without full Milvus.

\subsection{Steps}

\begin{enumerate}[leftmargin=*]
  \item Clone \textbf{milvus-io/knowhere} pinned to your Milvus release (e.g.\ v2.5.x).
  \item Study CMake options for GPU / cuVS / RAFT in the NVIDIA build docs.
  \item Fork or patch: link \texttt{hipVS} / \texttt{hipRAFT} instead of \texttt{cuvs} /
        \texttt{raft}; add backend flag (e.g.\ \texttt{KNOWHERE\_WITH\_HIP} vs
        \texttt{KNOWHERE\_WITH\_CUVS}).
  \item Run Knowhere unit tests for one index, e.g.\ \texttt{GPU\_IVF\_FLAT}:
        build on GPU, search with \texttt{nprobe} sweep, compare recall curve to CPU
        \texttt{IVF\_FLAT} on the same data.
\end{enumerate}

\textbf{Pass:} Knowhere tests green; recall vs \texttt{nprobe} rises like CPU IVF (not
necessarily bit-identical).

This is where ``replace cuVS/cuRAFT'' actually happens---in Knowhere's build and GPU index
wrappers, not in \texttt{run\_milvus\_hdf5.py}.

\section{Layer 3 --- Milvus from source (end-to-end)}

\subsection{Goal}

Milvus server uses the HIP-enabled Knowhere build on AMD GPU.

\subsection{Steps}

  \item Run Standalone with GPU devices visible (example):

  \item Use Milvus GPU index params (\texttt{cache\_dataset\_on\_device}, etc.; see Milvus
        GPU index docs for your version).
\end{enumerate}

\textbf{Pass:} \texttt{create\_index}, \texttt{load}, and \texttt{search} succeed; logs show
GPU index path (not CPU SSE-only Knowhere).

Extend the harness minimally:
\begin{itemize}[leftmargin=*]
  \item Add \texttt{--index-type GPU\_IVF\_FLAT} and GPU search params (\texttt{nprobe},
        device cache flags).
  \item Keep the same SIFT-1M, same \texttt{nprobe} grid ($1,4,8,16,32$), same $k=10$.
  \item Log: index build time, GPU memory (\texttt{rocm-smi}), recall@10, QPS.
\end{itemize}

\textbf{Compare:}
\begin{enumerate}[leftmargin=*]
  \item Recall vs \texttt{nprobe} must rise like FAISS (correctness).
  \item QPS vs \texttt{nprobe}---GPU should beat CPU Milvus at some operating points
        (performance).
  \item Index build time---often where GPU wins first.
\end{enumerate}

\section{Engineering workflow}

Recommended flow:

\begin{enumerate}[leftmargin=*]
  \item Layer~0: ROCm + Milvus CPU healthy on \texttt{amd-rx7900xtx}.
  \item Layer~1: hipVS/hipRAFT standalone on \texttt{gfx1100}.
  \item Layer~2: Knowhere fork + HIP link; one index type.
  \item Layer~3: Milvus source build; Standalone GPU search smoke test.
  \item Layer~4: \texttt{ann-harness-amd} SIFT-1M \texttt{nprobe} sweep vs CPU Tables~5--6.
  \item If recall OK and GPU faster at target points: widen index types + VectorDBBench GPU
        cases.
  \item If not: debug numerics, params, device cache before advancing.
\end{enumerate}

\textbf{Branching:} maintain a \texttt{milvus-amd-gpu} fork (Knowhere submodule + CMake
patches). Pin versions: Milvus tag $\leftrightarrow$ Knowhere tag $\leftrightarrow$ hipVS
release $\leftrightarrow$ ROCm version in a single compatibility table per experiment.

\textbf{CI/local:} small dataset on every commit; full SIFT-1M nightly on the lab host in
\texttt{tmux}.

\section{Scope and realism}

\begin{itemize}[leftmargin=*]
  \item \textbf{hipVS is early} (e.g.\ v0.1.x)---expect gaps vs cuVS; start with
        \texttt{GPU\_IVF\_FLAT} or GPU brute force, not every index at once.
  \item \textbf{API compatibility $\neq$ drop-in binary}---Milvus/Knowhere still need
        build-system and wrapper changes.
  \item \textbf{CPU baseline stays essential}---GPU work asks whether the hipVS path improves
        the recall--QPS trade-off vs CPU Milvus/FAISS on the same host, not whether Milvus
        beats FAISS on raw in-process QPS.
  \item \textbf{Stock Docker Milvus} remains the CPU control; GPU porting is source-build +
        fork work, not a config toggle.
\end{itemize}
